Introduction¶

Dataset Description¶

WeRateDogs @dog_rates is a Twitter account that rates people's dogs with a humorous comment about the dog. These ratings almost always have a denominator of 10. The numerators, though? Almost always greater than 10. 11/10, 12/10, 13/10, etc. Why? Because "they're good dogs Brent." WeRateDogs has over 4 million followers and has received international media coverage.

Question for Analysis¶

What stage of dog had the highest retweet and favorite counts.

Gathering Data¶

The libraries for the wrangling and analysis of this data were first imported. The data from the different sources were then gathered. Twitter enhanced archive data was provided in the csv format, while twitter image predictions file was provided via the link (https://d17h27t6h515a5.cloudfront.net/topher/2017/August/599fd2ad_image-predictions/image-predictions.tsv) and this was downloaded programmatically and read into the jupyter notebook. I was denied access to a twitter developer account and I had to use the twitter data (json) provided by Udacity and this was read into a data frame for easy analysis.

Assessing Data¶

Assessment of the data was done visually using Microsoft excel the various commands in Jupyter notebook like .head(),.sample(), .info(), .describe(). Assessing the various tables, twitter_id is duplicated in the twitter archive and twitter image prediction datasets. In the twitter data set twitter_id is represented as id and this needs to be corrected for easy merging of the datasets.

Wrangling the Data¶

Cleaning 1 The id in the tweet data was changed to tweet_id so it can be merged with twitter archive and twitter image prediction. Missing data

Tidiness of the data 1 The three dataframes were merged into one and it was called clean_tweet_arch. 2 The stages of dogs in four columns were merged into one column for easy analysis.

Quality of the data 1 The tweet_id was then changed to string instead of integer. 2 The timestamp was changed to datetime from string. 3 The rating_numerators of the twitter archive had some values that were less than 10 and greater than 20. 4 The rating_denominators had values that were not 10. These had to be removed. 5 Columns 'in_reply_to_status_id', 'in_reply_to_user_id', 'retweeted_status_id', 'retweeted_status_user_id', and 'retweeted_status_timestamp' were removed from the dataset. 6 Names having a, o, none etc were changed to null. 7 Duplicated jpg_url were also removed from the data set. 8 Underscores were removed from p1, p2, p3 columns. 9 Names in p1, p2, p3 starting with lowercase were changed to uppercase.

Storing Data¶

The clean_tweet_arch was then saved as twitter_archive_master.csv after cleaning.

Conclusion¶

From our analyses and visualization, we can see that the pupper had the highest value count and the least favorite and retweet count. While floofer had the least value count and still had the favorite and retweet count. We can also see that those who had high favorite count also had high retweet count, this signifies a direct relationship between these two.
